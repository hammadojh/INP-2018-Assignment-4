How to design a usable user interface

Today user intefaces have become all around us. Designing a usable user interface is very important in order to make people's lives easier. A usable user intrface is a one that is effective, effecient and satisfactort to all users who use it. In this pos I will give some tips in how to make the UI usable.

Know your users

The first step is to know your users very well. Imaging that you are designing a website for kids. Obviously it will be different from designing it for adults, for instance. As long as you gather more data about your users, it will be help you more in making your user intrface more usable to them.

Kwoing that, what the things that you need to know about your users. The following are some examples of what you need to gather about users :

  • age
  • gender
  • background
  • preferences
  • abilities/disabilities

a well known approach for user interface designers is to create what is called a persona . A persona contains information about an imaginery user representing one segment of the targeted users. the following image is an example of a persona.

Prototype, Test, Repeat

Before digging into code, prototype your user intrface. In this phase you should focus on layouting the main elements of your ui. Sketch your screens in a well structured way. You shouldn't focus on the visual design for now.

After you are done with your prototype, find real users and test it. By doing so, you will defenitely find some problems in your ui. Users will find some tasks confusing. Others will not know how to perfrom them. So, it is time to fix these issues. Go back to your prototype, fix it and test it again.

Gather usage information

After you are done will prototyping your are going to design and implement your app or website. Are you done by doing so? Of course not. You will start gathering what is called Usage Data . The following are some examples of usage data that you need to collect.

data type definition
Exit points these are the screens where users exit your website from. you need to know them to know what are the causes for people to exit.
Session time the time the user spend using your website/app per session. Sometimes you need people to spend little time and sometimes the opposite. So, knowing this will help.
Bounce rate The rate of which users enters your website/app then exits instantly without interacting with it. This piece of information is important because it indicates that there is a problem. The goal is to reduce this rate as much as we can.

Conclusion

Desiging a usable user inteface is very important. To achive that, a designer needs to know about his users. After that, prototyping and testing is an essential step till. This step is done many times till a level of satisfaction is reached. Finally, we need to collect usage data, like Exit points, Session Time and Bounce rate.
